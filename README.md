# ネットワーク基礎ハンズオン（第二弾）

## 📖 教材の概要

本教材は、**Dockerを使った開発経験を持つSIer若手エンジニア**が、通信トラブルに遭遇したときに「**なぜその通信は成立する／しないのか**」を構造的に説明できるようになることを目的とした、実務志向のネットワーク基礎ハンズオンです。

### 🎯 学習目標

受講者は最終的に以下を満たすことを目指します：

- **構成図を見て、通信経路を言語化できる**
- **通信失敗時に、以下の観点で切り分けができる**
  - ポート
  - ネットワーク境界
  - 名前解決
- **修正内容を「理由付き」で説明できる**

> 💡 正解は1つでなくてよい。「**説明できる**」ことを最重要視します。

### 🏗️ 教材の特徴

- **実務志向**: 実際のトラブルシューティングに直結するスキルを習得
- **体験重視**: 「動かない」状況を意図的に作り、解決過程を学習
- **段階的学習**: 5つのステップで基礎から応用まで体系的に学習
- **アナロジー活用**: 複雑な概念を身近な例で理解しやすく説明

---

## 📚 学習ステップ

### Step 0: 全体像とゴールの共有
- 学習対象の限定とゴールの明確化
- 正常系・異常系構成図の理解

### Step 1: ポートの役割を再確認する
- 「どこで待ち受けているか」の意識
- 待ち受けと接続は別概念であることの理解

### Step 2: Dockerによる視点の分離
- ホスト視点とコンテナ視点の違い
- 「どの視点で見ているか」の言語化

### Step 3: Dockerネットワークの分離（最重要）
- ネットワーク境界の存在を体感
- 「同じマシン＝同じネットワークではない」の理解

### Step 4: 名前解決の役割を理解する
- Docker内蔵DNSの仕組み
- サービス名とIPアドレス変換の理解

### Step 5: 総合演習（切り分け）
- 複合的な問題の段階的解決
- 実務レベルのトラブルシューティング手法

---

## 🚀 使用方法

### 前提条件
- Docker Desktop がインストール済み
- 基本的なDockerコマンドの理解
- `git` コマンドの基本操作

### 学習の進め方

1. **リポジトリをクローン**
   ```bash
   git clone https://github.com/18uma/infra-handson-docker-network.git
   cd infra-handson-docker-network
   ```

2. **各ステップのブランチに移動して学習**
   ```bash
   # Step 0から開始
   git checkout step0-base
   # STEP0_README.md を読んで実習
   
   # 次のステップへ
   git checkout step1-port
   # STEP1_README.md を読んで実習
   
   # 以下同様に進める...
   ```

3. **各ステップの README を読みながら実習**
   - 各ブランチには対応する `STEPX_README.md` があります
   - 手順に従って実際にコマンドを実行してください
   - 「なぜそうなるのか」を考えながら進めてください

### ブランチ構成

| ブランチ | 内容 | README |
|---------|------|--------|
| `main` | 完成形（講師・答え確認用） | 本README |
| `step0-base` | Step0: 全体像把握 | STEP0_README.md |
| `step1-port` | Step1: ポート理解 | STEP1_README.md |
| `step2-viewpoint` | Step2: 視点分離 | STEP2_README.md |
| `step3-network-isolation` | Step3: ネットワーク分離 | STEP3_README.md |
| `step4-dns` | Step4: 名前解決 | STEP4_README.md |
| `step5-final-exercise` | Step5: 総合演習 | STEP5_README.md |

## 🛠️ 使用技術

- **Node.js**: 通信確認用の最小API
- **React**: フロントエンドアプリケーション

### 非対象

- プロトコル内部仕様
- パケット解析

## 📈 学習効果

本教材を完了すると、以下のスキルが身につきます：

### 技術スキル
- **ネットワーク境界の理解**: 見えない境界の存在と影響
- **DNS名前解決**: Docker内蔵DNSの仕組み
- **ポート設定**: 待ち受けと接続の概念
- **ログ分析**: エラーメッセージからの原因特定

### 思考スキル
- **構造的思考**: 通信経路の論理的な分析
- **仮説検証**: 問題の原因を推測し検証する能力
- **優先順位判断**: 効率的な問題解決順序の決定
- **説明力**: 技術的な内容を理由付きで説明する能力

### 実務応用
- **障害対応**: 本番環境でのトラブルシューティング
- **設計レビュー**: ネットワーク設計の妥当性判断
- **環境構築**: 開発・テスト環境の構築とトラブル解決
- **チーム支援**: 他のメンバーへの技術的サポート
